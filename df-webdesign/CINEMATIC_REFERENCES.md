# Cinematic Reference Protocol — DF Webdesign

## CORE RULE

I will continuously receive cinematic reference videos, concept clips, motion references,
environment references, camera movement references, and visual inspiration material.

Every uploaded video MUST be analyzed and used as a core creative direction reference.

References are NOT optional. They define the project's visual language.

---

## WHEN A REFERENCE VIDEO IS UPLOADED

Analyze it across all 10 dimensions before writing any code:

1. **Emotional atmosphere** — What feeling does it create? Awe? Dread? Wonder? Scale?
2. **Camera behavior** — Speed, weight, drift angle, lerp factor, micro-movement
3. **Scene composition** — Depth layers, focal points, negative space, scale contrast
4. **Movement speed** — Is it slow and heavy? Fast and aggressive? Breath-like?
5. **Environmental scale** — How big does the world feel? How small does the viewer feel?
6. **Cinematic timing** — Pause duration, transition timing, easing curve behavior
7. **Scene transitions** — Cut, dissolve, camera push, fog fade, light shift?
8. **Lighting and fog** — Hard vs. soft, volumetric vs. flat, color temperature, density
9. **Motion density** — How much is moving? How much is static? What is the rhythm?
10. **Spatial storytelling** — How does the environment tell a story without words?

---

## ANALYSIS OUTPUT FORMAT

For every reference, produce:

### Atmosphere
What emotional state does this reference create?

### Camera System
- Movement type (drift / push / orbit / float)
- Speed (very slow / slow / medium)
- Lerp factor equivalent (0.02 = cinematic slug / 0.08 = responsive)
- Micro-drift behavior
- Look-at behavior

### Lighting Profile
- Key light color and angle
- Fill light behavior
- Fog density and color
- Bloom/glow behavior
- Contrast level (low = atmospheric / high = dramatic)

### Motion Language
- Primary motion type (float / drift / pulse / stream / orbit)
- Speed scale
- Easing style
- Rhythm (continuous / breath-like / reactive)

### Web Translation
How to reinterpret the reference into the website — NOT copying, but preserving:
- The emotional core
- The spatial feeling
- The pacing
- The atmosphere

### Implementation Priority
Which scene benefits most from this reference?
What specific system should be updated first?

---

## WHAT REFERENCES CAN DEFINE

Each reference can update any of these systems:

| System | What can change |
|---|---|
| Camera path | Speed, lerp factor, drift angle, keyframe positions |
| Lighting | Colors, intensity, transition speed, fog density |
| Particles | Count, size, speed, color, behavior |
| Environment scale | Object sizes, Z-spacing between scenes |
| Motion easing | LERP factors, sine wave frequencies |
| Fog | near/far values, color, density |
| Post-processing | Bloom intensity/threshold, vignette darkness, grain |
| Scene timing | How long camera lingers, transition duration |
| Color palette | Accent colors, ambient colors, emissive colors |

---

## CREATIVE RULES FOR REFERENCE INTERPRETATION

### DO:
- Extract the emotional feeling
- Identify the single most important motion principle
- Find the lighting secret that makes it feel premium
- Find the pacing secret that makes it feel cinematic
- Translate the scale feeling into Z-axis spacing

### DO NOT:
- Copy the reference literally
- Use colors that break the black/blue/white palette
- Introduce motion that is faster than the current system
- Add complexity that kills performance
- Clone sci-fi aesthetics into cheap cyberpunk

---

## CONSISTENCY RULE

As references accumulate:
- Build a unified visual language from all references combined
- New references refine, they do not replace
- Maintain stylistic coherence across all 5 environments
- Each environment should feel like it belongs to the same universe

---

## REFERENCE INFLUENCE HIERARCHY

When references conflict in style:

1. **Atmosphere** always wins over visual complexity
2. **Slowness** always wins over speed
3. **Depth** always wins over decoration
4. **Minimalism** always wins over excess
5. **Camera feeling** always wins over object animation

---

## THE TARGET FEELING

Every reference should push the experience toward:

- **Interstellar** — Scale, silence, depth, cosmic weight
- **Apple cinematic motion** — Precision, luxury, restraint, perfect timing
- **Unreal Engine demos** — Environmental realism, volumetric depth, light behavior
- **Sci-fi operating systems** — Holographic clarity, functional beauty, spatial UI
- **A24 film aesthetics** — Emotional atmosphere, deliberate pacing, meaningful emptiness

The visitor should feel:
- **Awe** at the scale
- **Curiosity** to keep scrolling
- **Immersion** — forgetting they are on a website
- **Trust** — this is a premium, serious agency
- **Wonder** — this is unlike anything they have seen before
